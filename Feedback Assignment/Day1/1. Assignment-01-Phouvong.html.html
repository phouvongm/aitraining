Operation & Monitoring

ວຽກປະຈຳວັນ

ການ ຕິດຕາມ ແລະ ຮັບມື ທັນທີ — 4 ໜ້າວຽກ ຫຼັກ
1
ກວດສອບ ຄວາມ ປອດໄພ ແລະ ລະບົບ (Daily Health Check)
ດຳເນີນ ການ ຕິດຕາມ ລະບົບ ເຄືອຂ່າຍ (Network), Cloud Infrastructure ແລະ Server (AWS EC2, Windows Servers) ປະຈຳ ວັນ ເພື່ອ ຮັບປະກັນ ສະຖຽນລະພາບ ໃນ ການ ດຳເນີນ ງານ ແລະ ເຝົ້າ ລະວັງ ໄພຄຸກຄາມ ທາງ ໄຊເບີ (Cyber Security) ຢ່າງ ໃກ້ ຊິດ.
AWS EC2 Windows Server Network Cyber Security
2
ບໍລິຫານ ຈັດການ ທີມ Support
ກວດສອບ ຄວາມ ຄືບໜ້າ ຂອງ IT Helpdesk, ຕິດຕາມ ເຄດວຽກ (Tickets) ທີ່ ຄ້າງຄາ ຢ່າງ ເປັນ ປະຈຳ ແລະ ໃຫ້ ການ ສະໜັບສະໜູນ ທີມງານ ໃນ ການ ແກ້ໄຂ ບັນຫາ ເຕັກນິກ ທີ່ ມີ ຄວາມ ຊັບຊ້ອນ.
Helpdesk Tickets SLA Monitoring
3
ຄຸ້ມຄອງ ສິດ ຜູ້ໃຊ້ ງານ (User Access Control)
ດຳເນີນ ການ ອະນຸມັດ ແລະ ບໍລິຫານ ຈັດການ ບັນຊີ ຜູ້ໃຊ້ ງານ (User Access Control) ພ້ອມ ທັງ ລະບົບ ອີເມວ (Email Administrator) ໃຫ້ ແກ່ ພະນັກງານ ທີ່ ເຂົ້າ-ອອກ ໃໝ່ ໂດຍ ສອດຄ່ອງ ກັບ ນະໂຍບາຍ ດ້ານ ຄວາມ ປອດໄພ ຂອງ ອົງກອນ.
User Access Email Admin Security Policy
4
Brief ງານ ຕອນ ເຊົ້າ (Stand-up Meeting)
ປະຊຸມ ສັ້ນ ຮ່ວມ ກັບ ທີມງານ ໃນ ຕອນ ເຊົ້າ ເພື່ອ ແບ່ງ ງານ, ກຳນົດ ບູລິມະສິດ ໃນ ມື້ ນັ້ນ ແລະ ຮັບຟັງ ບັນຫາ ປະຈຳ ວັນ ທີ່ ທີມງານ ປະສົບ ເຫັນ ເພື່ອ ການ ປະສານ ງານ ໃຫ້ ມີ ປະສິດທິພາບ.
Daily Standup Team Alignment

Maintenance & Alignment

ວຽກປະຈຳອາທິດ

ການ ບຳລຸງຮັກສາ ແລະ ປະສານ ງານ — 4 ໜ້າວຽກ ຫຼັກ
1
ບຳລຸງຮັກສາ ລະບົບ (Weekly Maintenance)
ປະສານ ງານ ກັບ ຜູ້ ຊ່ຽວຊານ ພາຍນອກ ຫຼື ທີມງານ ພາຍ ໃນ ເພື່ອ ດຳເນີນ ການ ອັບເດດ Patch ດ້ານ ຄວາມ ປອດໄພ, Backup ຂໍ້ມູນ Server ແລະ ກວດສອບ ປະສິດທິພາບ ຂອງ Network ໃຫ້ ຢູ່ ໃນ ມາດຕະຖານ ທີ່ ກຳນົດ.
Patch Management Server Backup Network Performance
2
ທົບທວນ ຜົນງານ ຂອງ ທີມ
ກວດສອບ ຕົວ ຊີ້ວັດ (KPIs / SLA) ຂອງ ທີມ IT Helpdesk ປະຈຳ ອາທິດ ເພື່ອ ປະເມີນ ວ່າ ການ ໃຫ້ ບໍລິການ ເປັນ ໄປ ຕາມ ມາດຕະຖານ ທີ່ ກຳນົດ ໄວ້ ຫຼື ບໍ່ ແລະ ດຳເນີນ ການ ປັບປຸງ ຕາມ ຄວາມ ຈຳເປັນ.
KPI Review SLA Compliance Service Quality
3
ປະຊຸມ ວາງ ແຜນ ພາຍ ໃນ
ຈັດ ການ ປະຊຸມ ປະຈຳ ອາທິດ ຮ່ວມ ກັບ ທີມງານ ເພື່ອ ຕິດຕາມ ຄວາມ ຄືບໜ້າ ຂອງ ໂຄງການ ພັດທະນາ ລະບົບ ພ້ອມ ທັງ ໃຫ້ ຄຳ ປຶກສາ ແລະ ຄຳ ແນະນຳ ແກ່ ພະນັກງານ ພາຍ ໃຕ້ ການ ຄຸ້ມຄອງ.
Weekly Planning Project Tracking Team Coaching
4
ກວດສອບ ແຜນ ຄວາມ ປອດໄພ
ທົບທວນ ລາຍງານ ການ ກວດສອບ (Logs) ດ້ານ ຄວາມ ປອດໄພ ທາງ ໄຊເບີ ປະຈຳ ອາທິດ ເພື່ອ ວິເຄາະ ຄວາມ ສ່ຽງ ແລະ ປະຕິບັດ ມາດຕະການ ປ້ອງ ກັນ ລ່ວງ ໜ້າ ກ່ອນ ທີ່ ໄພຄຸກຄາມ ຈະ ສ້າງ ຄວາມ ເສຍຫາຍ.
Security Logs Risk Analysis Threat Prevention

ການ ວາງ ແຜນ ຍຸດທະສາດ ແລະ ສະຫຼຸບ ຜົນ — 4 ໜ້າວຽກ ຫຼັກ
1
ລາຍງານ ຜົນງານ ຕໍ່ ຄະນະ ບໍລິຫານ
ສ້າງ ບົດ ລາຍງານ ການ ດຳເນີນ ງານ (IT Infrastructure & Support Performance) ນຳ ສະເໜີ ຕໍ່ ຄະນະ ບໍລິຫານ ປະຈຳ ເດືອນ ເພື່ອ ສະແດງ ເຖິງ ຄວາມ ພ້ອມ ໃນ ການ ຮອງຮັບ ຍຸດທະສາດ EMI Pathway 2030 ຂອງ ສະຖາບັນ.
Board Report Performance EMI Pathway 2030
2
ບໍລິຫານ ງົບປະມານ ແລະ ການ ຈັດຊື້
ວາງ ແຜນ ງົບປະມານ ໄອທີ, ກວດສອບ ຄ່າ ໃຊ້ ຈ່າຍ Cloud (AWS) ປະຈຳ ເດືອນ ແລະ ວາງ ແຜນ ການ ຈັດຊື້ ອຸປະກອນ ໄອທີ ໃຫ້ ສອດຄ່ອງ ກັບ ຄວາມ ຕ້ອງການ ຂອງ ອົງກອນ ແລະ ກອບ ງົບປະມານ ປະຈຳ ປີ.
IT Budget AWS Cost Procurement
3
ທົດສອບ ແຜນ ກູ້ ຄືນ ລະບົບ (DRP Testing)
ດຳເນີນ ການ ທົບທວນ ແລະ ທົດສອບ Disaster Recovery Plan (DRP) ປະຈຳ ເດືອນ ເພື່ອ ຮັບປະກັນ ວ່າ ອົງກອນ ສາມາດ ດຳເນີນ ທຸລະກິດ ໄດ້ ຢ່າງ ຕໍ່ ເນື່ອງ ໃນ ກໍລະນີ ເກີດ ເຫດ ສຸກ ເສີນ ຫຼື ໄພ ພິບັດ.
DRP Testing Business Continuity RTO / RPO
4
ປະເມີນ ຜົນງານ ແລະ ພັດທະນາ ບຸກຄະລາກອນ
ດຳເນີນ ການ ປະເມີນ ຜົນງານ ປະຈຳ ເດືອນ ຂອງ ທີມງານ ພ້ອມ ທັງ ວາງ ແຜນ ການ ຍົກ ລະດັບ ທັກສະ (Upskilling) ດ້ານ Digital ໃຫ້ ແກ່ ພະນັກງານ ເພື່ອ ຕອບ ໂຈດ ການ ຫັນ ປ່ຽນ ສູ່ ດິຈິຕ້ອນ ຂອງ ສະຖາບັນ.
Performance Review Digital Upskilling Talent Development
